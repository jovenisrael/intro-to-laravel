Laravel + Blade CRUD Tutorial

Student Management System — same idea as plain PHP, using Laravel’s structure

Migrations for the schema, .env for configuration, controllers for logic, Blade for the frontend

What you’ll learn

  • Installing Laravel with Composer
  • What the .env file is for and which variables matter
  • Creating the database and running migrations (instead of pasting SQL only)
  • Eloquent model, controller actions, and Blade templates
  • How the model, controller, and view (Blade) split responsibilities
  • Listing students and creating new ones with validation and CSRF protection
  • Running the app with php artisan serve (or Apache pointing at public/)
  • Laravel is a MVC framework, which stands for Model, View, and Controller.
  • Laravel documentation is available at https://laravel.com/docs

Please note that every command should be run in terminal or command prompt.

1Prerequisites

You need PHP and MySQL available (XAMPP is fine). Laravel also requires the Composer dependency manager.

Check PHP

In a terminal (PowerShell or Command Prompt):

php -v

Laravel 10+ typically needs PHP 8.1 or newer. Enable common extensions (OpenSSL, PDO, Mbstring, Tokenizer, XML, Ctype, JSON, BCMath).

XAMPP (MySQL)

  1. Start Apache and MySQL from the XAMPP Control Panel.
  2. phpMyAdmin: http://localhost/phpmyadmin

2Install Composer, then Laravel

Install Composer

  1. Download the Windows installer from https://getcomposer.org/download/
  2. Run it and ensure it can find your php.exe (often under C:\xampp\php\php.exe).
  3. Verify in terminal or command prompt:
composer -V

Create a new Laravel project

From a folder where you keep projects (example: your user folder or next to htdocs): open terminal or command prompt and run the following commands:

composer create-project laravel/laravel student-crud
cd student-crud
What this does
  • create-project downloads Laravel and installs PHP packages into vendor/.
  • student-crud is your app folder name; change it if you like.

Application key

After copying .env (next section), generate the app key once:

php artisan key:generate

3The .env file

Laravel reads environment variables from .env at the root of the project. This keeps secrets and machine-specific settings out of your code.

First-time setup

  1. Copy the example file: .env.example → .env (Laravel’s installer may already do this).
  2. Run php artisan key:generate so APP_KEY is set (used for encryption and sessions).

Important variables (typical local XAMPP setup)

APP_NAME=StudentCRUD
APP_ENV=local
APP_KEY=base64:...          # set by key:generate — do not invent manually
APP_DEBUG=true              # show detailed errors in development only
APP_URL=http://localhost

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=student_crud
DB_USERNAME=root
DB_PASSWORD=
What each group is for
  • APP_* — application name, environment (local vs production), encryption key, debug mode, base URL.
  • DB_* — how Laravel connects to MySQL (used by migrations and Eloquent).
Security: Never commit real .env files to public repositories. Commit .env.example with placeholder values only. In production, set APP_DEBUG=false and use strong database credentials.

Config files under config/ (for example config/database.php) read these values via env('DB_HOST'), etc. After changing .env, you usually do not need to restart anything except your web server if applicable.

4Create the empty database

In phpMyAdmin, create a database named student_crud (same as DB_DATABASE in .env). You do not need to run the CREATE TABLE SQL by hand — the migration in the next step will create the students table.

5Migration (schema as code)

Migrations are PHP files that describe your tables. They are versioned, repeatable, and team-friendly.

Generate model + migration

php artisan make:model Student -m

This creates app/Models/Student.php and a new file in database/migrations/ like xxxx_xx_xx_xxxxxx_create_students_table.php.

Edit the migration

Open that migration file and use:

<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('students', function (Blueprint $table) {
            $table->id();
            $table->string('first_name', 50);
            $table->string('last_name', 50);
            $table->unsignedTinyInteger('age');
            $table->timestamps();
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('students');
    }
};
Notes
  • id() — auto-increment primary key (like your plain PHP tutorial).
  • timestamps() — adds created_at and updated_at (Laravel convention).
  • down() — reverses the migration if you run php artisan migrate:rollback.

Run migrations

php artisan migrate

6Laravel project structure (relevant parts)

student-crud/ ├── .env # environment config (not committed) ├── app/ │ └── Http/Controllers/ │ └── StudentController.php # request handling (like your crud/*.php) │ └── Models/ │ └── Student.php # Eloquent model ├── database/migrations/ # schema versions ├── resources/views/ │ └── students/ │ └── index.blade.php # HTML + Blade (frontend) ├── routes/web.php # URL → controller mapping └── public/ # web server document root
Compared to plain PHP
  • Migration replaces manual CREATE TABLE in phpMyAdmin (you still create the empty database).
  • Controller replaces standalone scripts like create_student.php.
  • Blade replaces mixed PHP/HTML in index.php with cleaner templates.
  • .env replaces hard-coded define('DB_...') in dbconn.php.

Model, view, and controller — when and how we use each

Laravel follows an MVC style: Model (data), View (what the user sees), Controller (glue between browser, model, and view). A simple flow: the browser hits a route → Laravel runs a controller method → the controller may use the model to load or save data → the controller returns a view (HTML) or a redirect (after a form submit).

Model — app/Models/Student.php

When we use it: Whenever we need to work with the students table: list rows, insert a row, update, delete, or later add relationships. One model class usually represents one database table.

How we use it in this tutorial: We do not put SQL in the Blade file. Instead, the controller calls Eloquent on the model, for example Student::orderBy('id')->get() to read all students and Student::create([...]) to insert. The model knows the table name (Laravel infers students from Student) and which columns are mass-assignable via $fillable.

Controller — app/Http/Controllers/StudentController.php

When we use it: For every web request you want to handle with your own logic: showing a page, accepting a form POST, returning JSON (API), and so on. Routes in routes/web.php point URLs to controller methods.

How we use it in this tutorial: index() fetches students through the model, then passes them to the view with return view('students.index', compact('students')). store() reads the HTTP request, validates input, creates a row via the model, then redirects back to the list with a flash message — it does not print HTML itself. Keeping requests, validation, and redirects in the controller keeps templates simple.

View (Blade) — resources/views/students/index.blade.php

When we use it: For anything visual: layout, tables, buttons, modals, forms. If it is markup the user sees in the browser, it belongs in a view (or partials included by a view).

How we use it in this tutorial: The view receives data only as variables (here, $students). It uses Blade syntax (@foreach, {{ $student->first_name }}) to render HTML. The form’s action posts to a named route that maps to store() — the view does not run database queries; it only displays data and collects input.

Quick rule of thumb
  • Model — database table + rules about that data (fillable, casts, relations).
  • Controller — “what happens for this URL / this form submit?” (orchestration).
  • View — “how does this page look?” (presentation only).

7Model and controller

The roles of the model and controller are summarized in section 6 — Model, view, and controller. Below is the code for this project.

app/Models/Student.php

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Student extends Model
{
    protected $fillable = [
        'first_name',
        'last_name',
        'age',
    ];
}

$fillable allows mass assignment for Student::create([...]) safely.

Create the controller

php artisan make:controller StudentController

app/Http/Controllers/StudentController.php

<?php

namespace App\Http\Controllers;

use App\Models\Student;
use Illuminate\Http\Request;

class StudentController extends Controller
{
    public function index()
    {
        $students = Student::orderBy('id')->get();

        return view('students.index', compact('students'));
    }

    public function store(Request $request)
    {
        $validated = $request->validate([
            'first_name' => ['required', 'string', 'max:50'],
            'last_name'  => ['required', 'string', 'max:50'],
            'age'        => ['required', 'integer', 'min:1', 'max:255'],
        ]);

        Student::create($validated);

        return redirect()
            ->route('students.index')
            ->with('success', 'Student created successfully!');
    }
}
How this maps to your old tutorial
  • index — same as selecting all rows and showing the table.
  • store — same as create_student.php: validate, insert, redirect with a message.
  • $request->validate() — server-side validation (complements HTML5 required).
  • Eloquent builds parameterized queries for you (similar goal to prepared statements).

8Blade view — resources/views/students/index.blade.php

This file is the view layer: presentation only. See when and how we use the view versus the model and controller.

Create the folder resources/views/students/ and add index.blade.php:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Students — Laravel CRUD</title>
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>
<div class="container">
    <h1 class="text-center p-5 bg-dark text-white mb-3">CRUD WITH LARAVEL + BLADE</h1>

    @if (session('success'))
        <div class="alert alert-success alert-dismissible fade show" role="alert">
            {{ session('success') }}
            <button type="button" class="btn-close" data-bs-dismiss="alert"></button>
        </div>
    @endif

    @if ($errors->any())
        <div class="alert alert-danger alert-dismissible fade show" role="alert">
            <ul class="mb-0">
                @foreach ($errors->all() as $error)
                    <li>{{ $error }}</li>
                @endforeach
            </ul>
            <button type="button" class="btn-close" data-bs-dismiss="alert"></button>
        </div>
    @endif

    <div class="d-flex justify-content-between mb-3">
        <h3>All Students</h3>
        <button type="button" class="btn btn-primary" data-bs-toggle="modal" data-bs-target="#studentModal">Create</button>
    </div>

    <div class="table-responsive">
        <table class="table table-bordered table-striped table-hover">
            <thead>
                <tr><th>ID</th><th>Name</th><th>Age</th><th>Action</th></tr>
            </thead>
            <tbody>
                @forelse ($students as $student)
                    <tr>
                        <td>{{ $student->id }}</td>
                        <td>{{ $student->first_name }} {{ $student->last_name }}</td>
                        <td>{{ $student->age }}</td>
                        <td></td>
                    </tr>
                @empty
                    <tr><td colspan="4">No students yet.</td></tr>
                @endforelse
            </tbody>
        </table>
    </div>
</div>

<!-- Modal -->
<div class="modal fade" id="studentModal" tabindex="-1" aria-labelledby="studentModalLabel" aria-hidden="true">
  <div class="modal-dialog">
    <div class="modal-content">
      <form action="{{ route('students.store') }}" method="POST">
        @csrf
        <div class="modal-header">
          <h5 class="modal-title" id="studentModalLabel">Add Student</h5>
          <button type="button" class="btn-close" data-bs-dismiss="modal" aria-label="Close"></button>
        </div>
        <div class="modal-body">
            <div class="mb-3">
              <label for="firstName" class="form-label">First Name</label>

            <div class="mb-3">
              <label for="age" class="form-label">Age</label>

            </div>
        </div>
        <div class="modal-footer">
          <button type="button" class="btn btn-secondary" data-bs-dismiss="modal">Close</button>
          <button type="submit" class="btn btn-primary">Save</button>
        </div>
      </form>
    </div>
  </div>
</div>

<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>
Blade highlights
  • @csrf — adds a hidden token; Laravel rejects POSTs without it (CSRF protection).
  • {{ }} — escaped output (helps prevent XSS).
  • route('students.store') — named route URL; if you change the path in web.php, the form still works.
  • old('first_name') — repopulates fields after validation errors.

9Routes — routes/web.php

<?php

use App\Http\Controllers\StudentController;
use Illuminate\Support\Facades\Route;

Route::get('/', [StudentController::class, 'index'])->name('students.index');
Route::post('/students', [StudentController::class, 'store'])->name('students.store');

GET / shows the list; POST /students handles the form (same flow as posting to create_student.php before).

10Run and test

Option A: Built-in server (simplest)

From the project root:

php artisan serve

Open http://127.0.0.1:8000

Option B: XAMPP Apache

Point a virtual host’s document root to student-crud/public, or symlink/copy carefully so the web root is always public/ — never expose the whole Laravel folder.

Checklist

  • MySQL running; database student_crud exists.
  • .env DB_* matches your MySQL user/password.
  • php artisan migrate completed without errors.
  • Create a student via the modal; see the success flash message and new row.
Troubleshooting: “Access denied” → check DB_USERNAME / DB_PASSWORD. “Base table or view not found” → run migrations. 419 on submit → missing @csrf or session/cookie issues.

Summary

You recreated the same student list + create flow using Laravel: Composer to install the framework, .env for DB and app settings, a migration for the students table, an Eloquent model, a controller for list/create logic, and a Blade view for the UI.

Next steps: Add edit/update and delete routes and controller methods; use Route::resource for a full REST-style CRUD; learn Form Requests for larger validation rules.
